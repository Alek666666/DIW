BDD - Présentation de HeidiSQL



HeidiSQL est un outil pour l'administration des BDD.

Depuis ce logiciel vous allez pouvoir créer, alimenter et interroger vos bases de données.



Installation de HeidiSQL

Pré-requis

  • Avoir installé un SGBDr (MySQL, MariaDB, etc.)



Si vous êtes sur Ubuntu (Linux) :

Accédez à Ubuntu Software, puis chercher "HeidiSQL", et installer le paquet.



Si vous êtes sur Windows :

Avec Laragon, HeidiSQL est pré-installé.
Une fois votre SGBDr démarré (MySQL ou MariaDB), l'outil est directement accessible depuis le Menu (clic droit) > MySQL ou MariaDB > HeidiSQL :


Si vous n'utilisez pas Laragon, rendez-vous sur la page de téléchargement du site de HeidiSQL pour télécharger l'installer 32/64 bit.

Exécutez le fichier et suivez les instructions pour installer HeidiSQL sur votre machine.



Paramétrage

Par défaut, les paramètres des bases de données sont :

  • nom du serveur : localhost
  • utilisateur : root@localhost (ou plus souvent, root)
  • mot de passe : aucun (ou 0000, ou 1234)

Si vous avez un utilisateur avec les droits administrateur, notamment sur Ubuntu (Linux), connectez-vous plutôt avec celui-ci !



Fonctionnement du logiciel

Pour commencer, observons l'interface :

  1. En haut à gauche, sur la fenêtre, vous retrouvez le chemin de la base et/ou de la table sélectionnée

  2. Dans le volet latéral de gauche, sont affichées toutes vos BDD

  3. Dans le volet principal, dans la barre de menu supérieure se trouvent différents onglets de navigation :

    • Hôte : permet de choisir l'adresse du serveur (ici, localhost)
    • Base de données : permet de se positionner sur la BDD voulue (ici, eval)
    • Table : permet de sélectionner une table sélectionnée (ici, client) ; on retrouve plus bas, dans l'encart Colonnes, les différentes rubriques de cette table
    • Données : liste les données de la table sélectionnée
    • Requête : affiche les requêtes en cours



Exercice : votre première base de données !

Dans l'onglet Requête, nous écrivons nos scripts de création et d’insertion de la base ainsi que toutes les requêtes permettant d’interroger la base.

En SQL, chaque instruction se termine par un point virgule ;.

Pour exécuter vos scripts, vous pourrez

  • soit cliquer sur le bouton avec l'icône Play dans la barre d'outils
  • soit appuyer sur la touche F9
  • pour exécuter une requête spécifique : clic droit sur la requête voulue et Exécuter cette requête

NB: Si vous ne voyez pas votre base : clic droit > Rafraîchir ou F5


Création de la base garage

Copiez-collez le script ci-dessous dans l'onglet Requête et exécutez-le :

DROP DATABASE IF EXISTS garage; 

CREATE DATABASE garage;  

USE garage; 

CREATE TABLE voiture ( 
    voit_id INT NOT NULL AUTO_INCREMENT PRIMARY KEY, 
    voit_marque VARCHAR(50) NOT NULL, 
    voit_prix INT 
); 

INSERT INTO voiture (voit_id, voit_marque, voit_prix) VALUES (1, 'Audi', 29990); 
INSERT INTO voiture (voit_id, voit_marque, voit_prix) VALUES (2, 'BMW', 8500); 
INSERT INTO voiture (voit_id, voit_marque, voit_prix) VALUES (3, 'Ford', 15550);


Rafraichissez la page (F5), puis observez les onglets :

  • Base de données : garage
  • Table : voiture
  • Données :


Attention : le script que vous écrivez dans l'onglet Requête ne sera pas sauvegardé au redémarrage de votre logiciel (mais vos bases seront toujours présentes) ;

Pour le moment, prenez l'habitude de créer vos fichiers dans votre IDE (Visual Studio Code par exemple) avec l'extension .sql, et de les enregistrer sur votre machine.

Il vous suffira alors de copier/coller votre script puis de l'exécuter.
